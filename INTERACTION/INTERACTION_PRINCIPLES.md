# CORE PRINCIPLES

## 1. Jangan Bikin User Mikir Terlalu Lama

Interaction harus:
- jelas
- cepat dipahami
- obvious

User tidak boleh:
- bingung harus klik apa
- bingung posisi mereka dimana
- bingung next action apa

Jika user harus berpikir terlalu lama:
= friction naik.

---

## 2. Satu Screen = Satu Fokus Utama

Jangan overload message.

Hindari:
- terlalu banyak CTA
- terlalu banyak button
- terlalu banyak informasi sekaligus

Prioritaskan:
- focus
- hierarchy
- readability

---

## 3. Minim Klik

Telegram bukan web app.

Setiap klik tambahan:
- menurunkan momentum
- menambah friction
- meningkatkan kemungkinan user keluar

Selalu cari:
- flow paling pendek
- navigation paling jelas
- action paling cepat

---

## 4. Momentum User Jangan Putus

Interaction harus terasa mengalir.

Hindari:
- dead-end
- flow dingin
- respons terlalu kaku
- interaction kosong tanpa arah

User harus selalu merasa:
- ada next step
- ada progress
- ada tujuan jelas

---

## 5. CTA Harus Jelas

User tidak boleh menebak:
- tombol mana yang penting
- langkah berikutnya apa
- action utama dimana

CTA utama harus:
- jelas
- singkat
- mudah dipahami

---

## 6. Navigation Harus Recoverable

User harus bisa:
- kembali
- membatalkan
- pindah menu
- recover dari salah klik

Jangan membuat flow yang menjebak user.

---

## 7. Interaction Harus Terasa Cepat

Walaupun proses backend lama:
user harus tetap merasa:
- system aktif
- system merespon
- interaction hidup

Gunakan:
- loading feedback
- progress feeling
- acknowledgement sederhana

---

## 8. Telegram Flow ≠ Mobile App Flow

Jangan memaksa pattern web/mobile app ke Telegram.

Telegram interaction harus:
- ringan
- cepat
- inline-first
- conversational

Fokus:
- chat momentum
- button clarity
- fast navigation

---

## 9. Hindari UI Berisik

Jangan:
- spam emoji
- terlalu banyak simbol
- terlalu ramai
- terlalu banyak dekorasi

Gunakan visual seperlunya.

Prioritas utama:
- readability
- hierarchy
- clarity

---

## 10. Interaction Harus Terasa Manusiawi

Bot tidak boleh terasa:
- dingin
- terlalu robotic
- terlalu formal
- terlalu corporate

Gunakan tone:
- akrab
- natural
- ringan
- mengalir

Tetapi:
tetap jelas dan profesional.

---

# INTERACTION PRIORITIES

Urutan prioritas interaction:

1. Clarity
2. Flow
3. Speed Feeling
4. Comfort
5. Retention
6. Visual Polish

Jika visual mengganggu clarity:
pilih clarity.

Jika animation/effect mengganggu speed:
pilih speed.

---

# ANTI-PATTERNS

Hindari:
- user bingung next action
- terlalu banyak button
- callback chaos
- nested navigation berlebihan
- text terlalu panjang tanpa hierarchy
- CTA ambigu
- interaction terlalu formal
- dead-end navigation
- spam edit message tanpa arah
- flow yang terasa berat

---

# FINAL DOCTRINE

Good interaction bukan tentang UI paling keren.

Good interaction adalah:
- user cepat paham
- user nyaman bergerak
- user tidak lelah
- user merasa flow natural
- user ingin kembali menggunakan bot
