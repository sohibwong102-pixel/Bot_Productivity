# RETENTION RULES

Project ini adalah AI Productivity Telegram Bot.
Semua keputusan system HARUS mempertimbangkan retention user.

---

# CORE RETENTION PRINCIPLES

## 1. Low Friction First
- minim langkah
- minim typing
- minim loading
- minim pindah menu
- minim dead-end UX

User harus bisa:
- buka bot
- klik
- langsung dapet value

dalam beberapa detik.

---

## 2. Instant Reward System
Setiap interaction harus memberi:
- hasil cepat
- feedback jelas
- rasa progress
- rasa “AI ngerti gue”

Hindari:
- waiting lama
- proses bertele
- UX kosong
- menu membingungkan

---

## 3. Clean Navigation
Navigation wajib:
- predictable
- simple
- inline keyboard oriented
- ada back navigation
- tidak nested terlalu dalam

Maksimal:
- 2–3 layer navigation

---

## 4. Telegram-Native UX
UX HARUS terasa native Telegram.

Prioritas:
- inline keyboard
- quick actions
- short response
- compact layout
- fast interaction

Hindari:
- paragraph panjang
- form panjang
- terlalu banyak command
- terlalu banyak setup

---

## 5. Retention > Complexity
Jika ada pilihan:
- fitur kompleks
VS
- UX simpel tapi engaging

pilih UX simpel.

---

## 6. AI Must Feel Useful
AI harus terasa:
- cepat
- membantu
- relevan
- tidak generic
- tidak robotic

Output harus:
- actionable
- clean
- langsung bisa dipakai

---

## 7. Progressive Engagement
Jangan expose semua fitur sekaligus.

Gunakan:
- progressive discovery
- guided interaction
- contextual recommendation

User harus merasa:
“oh ternyata ada fitur keren lagi”

---

## 8. Avoid Cognitive Overload
Jangan bikin user mikir terlalu keras.

Hindari:
- terlalu banyak pilihan
- terlalu banyak tombol
- terlalu banyak text
- layout berantakan

---

## 9. Mobile-First Thinking
Semua UX diasumsikan dipakai:
- layar kecil
- cepat
- sambil multitasking

Optimasi:
- vertical readability
- tap comfort
- fast scanning

---

## 10. Emotional UX
Bot harus terasa:
- hangat
- ringan
- helpful
- premium
- modern

Bukan:
- corporate
- kaku
- terlalu formal

Emoji boleh:
- sedikit
- elegan
- tidak norak

---

# SYSTEM DECISION PRIORITY

Saat memberi solusi:
1. retention
2. UX clarity
3. simplicity
4. maintainability
5. scalability

Bukan sebaliknya.

---

# ABSOLUTE RULES

JANGAN:
- overengineering
- multi-step flow tidak perlu
- refactor brutal tanpa alasan
- UX yang memperlambat user
- nested FSM rumit
- menu terlalu dalam
- callback chaos

SELALU:
- pikirkan friction
- pikirkan speed
- pikirkan emotional engagement
- pikirkan long-term retention
